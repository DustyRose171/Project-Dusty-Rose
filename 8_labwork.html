К концу занятия у каждого будет работающий публичный сайт собственного итогового проекта. Сайт сделан с помощью Quarto, исходники лежат в репозитории GitHub, публикация настроена через GitHub Pages.

Лабораторная состоит из восьми шагов. Каждый шаг — небольшое задание. Если что-то не получается — спрашивайте сразу, не копите вопросы.

Кстати, файл, который вы сейчас читаете — сам сделан в Quarto. Откройте его исходник, чтобы увидеть, как устроено то, что вы учитесь создавать.

1 Markdown: язык разметки

1.1 Зачем нужен Markdown

Когда вы пишете в Word, оформление и содержание перемешаны: жирный шрифт, отступы, цвета — всё это часть документа. Если открыть .docx блокнотом, увидите неразборчивую кашу из управляющих символов и текста.

Markdown — другой подход, это обычный текст с минимальной разметкой через специальные символы. Файл с расширением .md остаётся читаемым в любом редакторе. Программа-конвертер превращает разметку в красивый HTML, PDF или Word.

Чем Markdown хорош для исследователя:

  • читаемый исходник, который не зависит от программы
  • легко версионировать через Git (в отличие от Word)
  • легко публиковать в вебе
  • работает в GitHub, Telegram, Notion, Discord, Obsidian — везде

1.2 Базовый синтаксис

1.2.1 Заголовки

В Markdown шесть уровней заголовков. Каждый задаётся через решётки в начале строки:

# Заголовок первого уровня
## Заголовок второго уровня
### Заголовок третьего уровня

1.2.2 Абзацы и переносы

Абзац — это просто блок текста, отделённый пустой строкой сверху и снизу. Обычный перенос строки внутри абзаца Markdown игнорирует — текст склеивается.

Чтобы сделать принудительный перенос, поставьте в конце строки два пробела или обратный слеш \. Но обычно лучше начать новый абзац.

1.2.3 Выделения

*курсив* или _курсив_
**жирный** или __жирный__
***жирный курсив***
~~зачёркнутый~~
`код в строке`

Результат: курсив, жирный, жирный курсив, зачёркнутый, код в строке.

1.2.4 Списки

Маркированные списки — через дефис или звёздочку:

- Первый пункт
- Второй пункт
  - Вложенный пункт (отступ 2 или 4 пробела)
  - Ещё вложенный
- Третий пункт

Нумерованные — через цифру с точкой.

1. Первый
1. Второй
1. Третий

1.2.5 Ссылки и картинки

[Текст ссылки](https://example.com)
![Подпись к картинке](images/photo.jpg)

Картинка отличается от ссылки только восклицательным знаком впереди. Путь к картинке может быть локальным (относительно файла) или ссылкой в интернете.

1.2.6 Цитаты

> Это цитата.
> Может занимать несколько строк.
>
> Может содержать несколько абзацев.

1.2.7 Код

Однострочный код — через одинарные обратные кавычки: pd.read_csv().

Блок кода — через тройные обратные кавычки, можно указать язык для подсветки:

```python
import pandas as pd
df = pd.read_csv("data.csv")
```

1.2.8 Таблицы

| Колонка 1 | Колонка 2 | Колонка 3 |
|-----------|-----------|-----------|
| строка 1  | данные    | данные    |
| строка 2  | данные    | данные    |

Двоеточия в строке-разделителе задают выравнивание: :--- слева, ---: справа, :---: по центру.

1.2.9 Формулы

Quarto поддерживает LaTeX-формулы. В строке — через одинарные доллары, отдельным блоком — через двойные:

Формула в тексте: $E = mc^2$.

Формула блоком:
$$\text{CER} = \frac{I + D + S}{N}$$

2 Quarto: от Markdown к публикации

2.1 Что такое Quarto

Markdown — это синтаксис разметки. Quarto — это система публикации, которая принимает на вход .qmd-файлы (расширенный Markdown) и собирает из них сайты, статьи, презентации, книги, отчёты.

Главные возможности Quarto поверх обычного Markdown:

  • исполняемые блоки кода на Python, R, Julia
  • сборка многостраничных сайтов с навигацией
  • единая конфигурация оформления через YAML
  • встроенная поддержка цитирования, ссылок, нумерации
  • рендеринг в HTML, PDF, Word, ePub, презентации Reveal.js

2.2 Установка

ПредупреждениеПеред началом

Для следующих шагов нужны установленные Quarto и VS Code. Если ещё не установили — самое время сделать это сейчас.

Шаг 1. Quarto CLI. Идём на сайт https://quarto.org/docs/get-started/, скачиваем установщик для своей системы (Windows, Mac, Linux), запускаем. Установка обычная, без особенностей.

Проверка в терминале:

quarto --version

Должна вывести номер версии.

Шаг 2. VS Code и расширение Quarto. Если VS Code не установлен — скачиваем с https://code.visualstudio.com/. После установки открываем VS Code, идём во вкладку Extensions (Ctrl+Shift+X), ищем “Quarto”, устанавливаем расширение от Posit.

Расширение даёт подсветку синтаксиса, превью рендера и удобную работу с .qmd-файлами.

2.3 Первый рендер

Откройте папку dh-project в VS Code (File -> Open Folder). Файл index.qmd должен появиться в боковой панели.

В терминале VS Code (Terminal -> New Terminal) выполните:

quarto preview index.qmd

Команда соберёт страницу в HTML и откроет в браузере. Превью обновляется автоматически при сохранении файла — можно редактировать и сразу видеть результат.

2.4 Задание 2

Откройте свой index.qmd через quarto preview. Посмотрите, как выглядит написанное в Задании 1. Если что-то отображается не так, как ожидали, — поправьте исходник.

3 Вложения: картинки и интерактивные элементы

3.1 Картинки

Локальная картинка — относительный путь от файла:

![Подпись](images/photo.jpg)

Удалённая — полный URL:

![Подпись](https://example.com/photo.jpg)

Quarto расширяет базовый Markdown для картинок. Можно задать ширину, выравнивание, дать ссылку при клике:

![Подпись](images/photo.jpg){width=60% fig-align="center"}

3.2 Интерактивные элементы

На предыдущих семинарах вы делали интерактивные карты в folium и графы в pyvis. Каждый из них сохраняется как самостоятельный HTML-файл. Quarto умеет вставлять такие файлы как iframe — встроенные блоки внутри страницы.

<iframe src="my_map.html" width="100%" height="500" frameborder="0"></iframe>

Положите HTML-файл рядом с .qmd-файлом — и он встроится в страницу. То же самое работает для любых внешних HTML, видео, документов.

4 YAML: конфигурация через структурированный текст

4.1 Что такое YAML

YAML — формат записи структурированных данных в читаемом виде. Используется для конфигурации в большинстве современных инструментов: Docker, GitHub Actions, Quarto, и так далее.

Базовый синтаксис простой:

ключ: значение
другой_ключ: другое значение

вложенный_объект:
  внутренний_ключ: значение
  ещё_ключ: значение

список:
  - первый_элемент
  - второй_элемент
  - третий_элемент

Главные правила:

  • Иерархия задаётся отступами (обычно 2 пробела). Не табы — только пробелы.
  • Двоеточие после ключа, значение после пробела.
  • Списки — через дефис в начале строки.
  • Строки можно не заключать в кавычки (если в значении нет спецсимволов).

4.2 YAML в Quarto: front matter

Front matter — блок YAML в самом начале .qmd-файла, обрамлённый тройными дефисами:

---
title: "Заголовок страницы"
author: "Иванов И.И."
date: today
format:
  html:
    toc: true
    theme: flatly
---

Это указание самому Quarto: как называется страница, кто автор, как рендерить, какие настройки оформления применить. Конкретно эти настройки работают на уровне отдельного файла.

4.3 YAML в Quarto: конфигурация сайта

Когда у вас несколько связанных страниц, удобнее задать общие настройки в одном файле — _quarto.yml в корне проекта. Разберём пример с сайта нашего курса:

project:
  type: website
  output-dir: docs

website:
  title: "DH Tools"
  description: "Цифровые технологии в профессиональной деятельности"
  repo-url: https://github.com/always1ready/dh-tools
  navbar:
    background: white
    left:
      - href: index.qmd
        text: Главная
      - href: schedule.qmd
        text: Расписание
      - href: resources.qmd
        text: Ресурсы
    right:
      - icon: github
        href: https://github.com/always1ready/dh-tools
  page-footer:
    center: "DH Tools — НИУ ВШЭ, 2026"

format:
  html:
    theme: flatly
    toc: true
    lang: ru

Разбираем по блокам:

  • project — тип проекта (website, book, manuscript) и куда складывать собранные файлы (docs)
  • website — настройки сайта: заголовок, ссылка на репозиторий, навигационное меню (navbar), подвал (page-footer)
  • navbar.left и navbar.right — пункты меню слева и справа; каждый пункт — это объект со ссылкой и текстом
  • format — настройки рендера для всех страниц сайта: тема оформления, оглавление, язык

4.4 Примеры реальных Quarto-сайтов

5 GitHub: публикация в интернете

5.1 Концепции

Git — система контроля версий. Отслеживает все изменения в файлах: что, когда, кто изменил. Сегодня мы пользуемся ей опосредованно, через веб-интерфейс GitHub.

GitHub — сайт, на котором хранятся репозитории. Бесплатный для публичных проектов.

Репозиторий — папка проекта вместе с историей всех изменений.

Коммит — снимок состояния файлов в определённый момент. Каждое изменение оформляется как коммит с описанием.

GitHub Pages — бесплатный хостинг статических сайтов прямо из репозитория. Идеально подходит для Quarto-сайтов.

5.2 Шаг 1. Создание аккаунта и репозитория

Уведомление

Если у вас уже есть аккаунт GitHub — пропустите создание.

Идите на https://github.com, нажмите Sign Up, регистрируйтесь. Используйте университетскую почту, если хотите бесплатные расширения для студентов через GitHub Education.

После входа — справа вверху плюс -> New Repository.

  • Repository name: dh-project (или любое название латиницей через дефис)
  • Description: короткое описание (необязательно)
  • Public — обязательно, иначе GitHub Pages не сработают на бесплатном аккаунте
  • Add README file — отметить галочкой
  • Add .gitignore — пропустить
  • License — пропустить

Нажмите Create Repository.

5.3 Шаг 2. Загрузка файлов

Готовый репозиторий покажет страницу с одним файлом README.md. Сверху есть кнопка Add file -> Upload files.

Перетащите в окно весь содержимое папки dh-project:

  • index.qmd
  • _quarto.yml
  • папку docs/ (со всеми её файлами)
  • папку images/ (если есть)
Важное уведомление

Важно загрузить именно содержимое docs/, потому что GitHub Pages берёт сайт оттуда. Без этого опубликуется только исходник, а не готовый сайт.

Внизу страницы — поле Commit changes. Напишите в Commit message короткое описание: Add project files. Нажмите Commit changes.

5.4 Шаг 3. Настройка GitHub Pages

В репозитории идите в Settings (верхняя панель) -> Pages (левое меню).

В разделе Build and deployment установите:

  • Source: Deploy from a branch
  • Branch: main / /docs

Нажмите Save.

GitHub соберёт сайт. Через 30–60 секунд обновите страницу — сверху появится сообщение с публичной ссылкой формата:

https://ваш-логин.github.io/dh-project/

Откройте — это ваш сайт в интернете.

5.5 Шаг 4. Редактирование через веб

Любой файл в репозитории можно редактировать прямо на GitHub: открыть файл -> иконка карандаша справа сверху. После сохранения GitHub Pages пересоберёт сайт автоматически (через 30–60 секунд).

Но имейте в виду: вы редактируете только исходник .qmd, а GitHub Pages показывает уже собранный HTML из docs/. То есть после правки .qmd нужно либо локально перерендерить через quarto render и снова загрузить docs/, либо настроить автоматическую сборку через GitHub Actions (это можете разобрать дома).

5.6 Задание 5

Пройдите все четыре шага. К концу у вас должна быть публичная ссылка на сайт. Скиньте ссылку в общий чат — посмотрим работы друг друга.

6 Что дальше

СоветКуда расти

То, что вы сделали сегодня — минимальная рабочая основа. Дальше можно:

  • Добавить несколько страниц (разделы проекта)
  • Настроить кастомное оформление через styles.css
  • Подключить GitHub Actions для автоматического рендера при каждом коммите
  • Изучить локальный git (git clone, git commit, git push) для удобной работы из терминала
  • Добавить функцию поиска по сайту, цитирование, библиографию

Для итогового проекта по курсу вам может понадобиться сайт с несколькими страницами: описанием проекта, методами, результатами, исходным кодом. Сегодняшняя основа — стартовая точка для этого.
